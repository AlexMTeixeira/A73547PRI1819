    Resumo:

    Para a criação do servidor, foi necessário incluir três pacotes, http, url e fs. Sendo que, o pacote http serviu para criar o servidor e coloca-lo á escuta numa determinada porta. O pacote url teve como objetivo tratar e analisar o url passado pelo cliente. Por fim, o pacote fs (File System) veio responder ao cliente, com os ficheiros do Arquivo Musical Digital, criados na aula.

    Como forma de tratar o url, foi feita um destinção no pathname. Para isso, sempre que o pathname for "/manifesto", o File System é responsavel por transmitir o ficheiro manifesto-tp4.html. Caso o pathname seja "/obra", vão existir dois tipos de resposta.

    No caso, pathname igual a "/obra", verifica-se se existe uma query em que o seu codigo é id. Caso se verifique, o fs irá responder com o html correspondente à obra com o respetivo id. Caso contrário, o fs irá responder com o index do Arquivo Musical Digital.

    Resultados:

    • Pasta que contêm o website
    • XML com o conteudo do manifesto.
    • Schema de conversão XML para HTML.
    • Servidor em JavaScript.
